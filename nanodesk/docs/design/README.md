## 如何添加设计文档

当需要设计一个大型功能点时：

1. **创建文档**: 复制 [TEMPLATE.md](./TEMPLATE.md)（如有）或创建新文件
2. **命名规范**: `FEATURE_NAME.md`，大写下划线分隔
3. **必填内容**:
   - 背景与目标（解决什么问题）
   - 设计方案（如何解决问题）
   - 实施路线图（分阶段交付）
   - 验收标准（怎么算完成）
4. **更新索引**: 在本文件添加条目
5. **发起讨论**: 在 PR 中讨论，达成共识后标记为 🚧

---

## 与 TODO.md 的关系

- **TODO.md**: 短期任务清单，颗粒度小（1-2 天）
- **design/**: 大型功能设计，颗粒度大（1-4 周）

**工作流**:
```
设计文档 (design/) 
       ↓ 确定方案，拆解任务
TODO.md 任务项
       ↓ 开发完成
CHANGELOG.md 记录
```
